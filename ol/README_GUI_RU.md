# ETA Predictor — GUI (Tkinter) + сборка в EXE

Ниже — готовый набор файлов:
- `eta_regression_core.py` — ядро (обучение/прогноз/выгрузка Excel)
- `eta_gui_tk.py` — графический интерфейс (Tkinter)
- `eta_regression_cli.py` — CLI (как раньше, но через core)
- `requirements.txt` — зависимости
- `build_exe_windows.bat` — сборка `ETA_Predictor.exe` через PyInstaller

## 1) Как пользоваться GUI без EXE (если на ПК есть Python)
1. Установите Python 3.10+ (желательно 3.11) с галочкой **Add Python to PATH**.
2. В папке с файлами создайте окружение и установите зависимости:

```bat
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

3. Запуск GUI:

```bat
.venv\Scripts\activate
python eta_gui_tk.py
```

### Чтобы запускать двойным кликом (без консоли)
**Вариант A:** переименуйте `eta_gui_tk.py` в `eta_gui_tk.pyw` и запускайте двойным кликом.

> `*.pyw` в Windows запускается через `pythonw.exe` (без окна консоли), но Python и зависимости всё равно должны быть установлены.

**Вариант B:** сделайте ярлык на команду:
`pythonw.exe <путь_к_папке>\eta_gui_tk.py`

## 2) Как собрать EXE (рекомендуемый вариант для пользователей без Python)
На компьютере “сборщике” (у вас/у ИТ):
1. Положите все файлы в одну папку.
2. Запустите `build_exe_windows.bat` двойным кликом.
3. Готовый файл будет:

`dist\ETA_Predictor.exe`

Дальше его можно раздать пользователям. Python на их ПК не нужен.

## 3) Логика работы приложения
- Пользователь выбирает (или перетаскивает) входной `.xlsx`
- Нажимает **Запустить**
- На выходе появляется новый Excel (по умолчанию рядом), с:
  - красивым оформлением (шапка, фильтры, полосатые строки, ширины)
  - датами без времени (`DD.MM.YYYY`)
  - числовым столбцом `Остаток_дней` (не дата!)
  - пустыми предиктами для строк, где уже заполнен `Факт_приход_конечный`

`Стадия_модели` показывает, какая стадия (stage1/2/3) была использована для `ETA_модель`.

## 4) Если вы ведёте файл дальше
Можно работать прямо в `...with_pred.xlsx`: добавлять новые строки и дополнять факты. 
GUI при следующем прогоне обновит/пересчитает прогнозные колонки.
