# Пакет Пакетыч — сайт бренда подарочной упаковки

B2B-сайт-витрина с каталогом в формате маркетплейса, регистрацией пользователей и системой заявок менеджеру. Построен по согласованной концепции (витрина + маркетплейс + регистрация + заявки). **Без интеграции 1С** — она добавляется отдельным этапом.

## Технологии

- **Next.js 14** (App Router, SSG) — статическая генерация всех страниц товаров, отличный SEO
- **React 18 + TypeScript**
- **Tailwind CSS** — дизайн-система в тёплой «бумажной» эстетике бренда
- **Framer Motion** — анимации витрины (hero, появление секций, hover-эффекты)

## Структура

```
src/
  app/
    page.tsx                 Главная — витрина бренда
    home-client.tsx          Анимированные секции главной
    catalog/                 Каталог-маркетплейс (фильтры, поиск, сортировка)
      new/                   Новинки
      sale/                  Распродажа
    product/[slug]/          Карточка товара
    actions/                 Акции
    register/  login/        Регистрация и вход
    cart/                    Корзина заявки
    account/                 Личный кабинет (история заявок)
    favorites/               Избранное
    about/ wholesale/ contact/ privacy/
  components/                Header (умный поиск), Footer, ProductCard
  providers/                 AuthProvider, StoreProvider (избранное + заявка)
  lib/                       types, products, utils
  data/products.json         240 товаров (база пакетов + новогодняя коллекция)
public/
  brand/                     Логотип, маскот
  products/bags/             ~220 фото пакетов
  products/newyear/          Новогодняя коллекция (single + collection sheets)
```

## Локальный запуск

```bash
npm run build    # продакшен-сборка
```

## Деплой на Vercel

1. Создайте репозиторий на GitHub и запушьте проект:
   ```bash
   git init
   git add .
   git commit -m "Пакет Пакетыч — сайт v1"
   git branch -M main
   git remote add origin https://github.com/ВАШ_АККАУНТ/paket-paketych.git
   git push -u origin main
   ```
2. Зайдите на vercel.com → **Add New Project** → импортируйте репозиторий.
3. Vercel сам определит Next.js. Нажмите **Deploy** — ничего настраивать не нужно.
4. (Опционально) добавьте переменные окружения из `.env.example` в настройках проекта.

## Как сейчас работают данные (демо без бэкенда)

Чтобы сайт работал на Vercel сразу, без базы данных:

- **Пользователи, заявки, избранное** хранятся в `localStorage` браузера.
- Регистрация/вход — клиентские, для демонстрации сценария.
- Отправленная заявка сохраняется локально и видна в личном кабинете.

Это **временное** решение для макета. Логика полностью повторяет финальную, поэтому замена на реальный бэкенд не потребует переделки интерфейса.

## Что подключить перед боевым запуском

1. **Реальный бэкенд для пользователей и заявок.** Рекомендуется: PostgreSQL (через Supabase) + NextAuth + Prisma. Заменяются только провайдеры `AuthProvider` и обработчик отправки заявки в `cart/page.tsx`.
2. **Уведомления менеджеру.** При отправке заявки — POST на API-роут, оттуда email (Resend) и/или Telegram-бот.
3. **Интеграция с 1С ERP** (отдельный этап):
   - 1С → сайт: синхронизация флага наличия (`isAvailable`) и базовых цен.
   - сайт → 1С: передача заявок для формирования КП.
   - Сейчас поле `isAvailable` уже заложено в данные и управляет видимостью в поиске «только в наличии».
4. **CMS для контента** (опционально). Если нужно управлять каталогом без программиста — Sanity.io, как описано в ТЗ.
